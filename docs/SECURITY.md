# Модель безопасности

Защищаемые данные — TOTP secret и связанная метаинформация. У проекта нет
собственного сервера, аккаунтов, облачной синхронизации, аналитики или
crash-reporting.

## Android

Содержательная часть каждой записи шифруется AES-256-GCM с отдельным 12-byte IV.
IV генерирует Android Keystore provider. В SQLite открыто хранятся только ID,
revision, schema version, IV и ciphertext с GCM tag; display name, issuer, account
и secret входят в зашифрованный payload.

Неэкспортируемый ключ Android Keystore используется без биометрии и PIN приложения.
Сначала запрашивается StrongBox, при его недоступности используется обычный Android
Keystore. На API 35+ ключ доступен только при разблокированном устройстве.

Потерянный или невалидный ключ не заменяется автоматически при существующей БД.
Пользователь получает предупреждение и может явно удалить локальную БД и ключ.
Сброс Android-хранилища не удаляет уже синхронизированные данные с часов.

Backup/data extraction и screenshots/recents отключены. Release APK имеет
`debuggable=false` и удаляет транзитивные `INTERNET` и `ACCESS_NETWORK_STATE`.
Приложение не запрашивает `CAMERA`, storage, location или Bluetooth permissions.

Google Code Scanner показывает системный экран Google Play services. Приложение
получает строку QR, разбирает её локально и не отправляет её на собственный сервер.

## Garmin и транспорт

Garmin хранит секреты в `Application.Storage` без дополнительного прикладного
шифрования. Payload Garmin Mobile SDK также не шифруется приложением и использует
существующее сопряжение Garmin Connect. Checksum протокола обнаруживает случайное
повреждение, но не является MAC и не обеспечивает аутентичность.

Подтверждаемая очистка часов логически удаляет active/staging snapshot и favorite
через Connect IQ API. Ни Android, ни Garmin не гарантируют физическое стирание
данных из flash.

## Логи и временные данные

Секреты, коды, QR payload, расшифрованные записи и полные sync messages запрещено
помещать в logs, clipboard, exceptions, saved state или crash artifacts. Android
release logging отключён; debug Android и Garmin выводят только безопасные
диагностические категории.

Временные массивы secret очищаются там, где это практически возможно.

## Границы защиты

Android Keystore защищает ключ от экспорта, но код, уже выполняющийся в процессе
приложения на разблокированном устройстве, может запросить расшифрование. Debug APK,
rooted/скомпрометированное Android-устройство и доступ владельца через Garmin
developer tools не считаются защищённой средой.

При потере телефона или часов следует отозвать или перевыпустить TOTP credentials
у каждого issuer.

Signing keys хранятся вне репозитория с правами `0600` и отдельными защищёнными
резервными копиями. Их потеря не раскрывает TOTP secrets, но исключает выпуск
совместимых обновлений.
